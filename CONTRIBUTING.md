# Contributing to RiCo

Thank you for your interest in RiCo.

RiCo is an early-stage runtime governance architecture developed by ManChine AI Technology. Contributions are welcome when they help clarify, test, document, or improve the architecture without altering its core terminology or governance boundaries.

---

## Locked Terminology

The following terms are locked:

- **RiCo** = Runtime Integrity Control
- **REB** = Runtime Execution Boundary
- **CARE/REB** = Runtime Execution Boundary architecture, depending on context
- **Official slogan** = Humans First. Continuity Always. Architecture Must Survive.

Do not rename, reinterpret, or replace these terms in contributions.

---

## Architectural Boundary

RiCo governs whether execution remains admissible at runtime.

RiCo does not replace model logic, application logic, observation systems, policy engines, or human decision-making structures.

Contributions should preserve the distinction between:

- Observation
- Evaluation
- Governance
- Execution

---

## Contribution Guidelines

Contributions may include:

- Documentation improvements
- Runtime scenarios
- Architecture examples
- Governance boundary clarification
- Terminology consistency fixes
- Diagrams or explanatory materials

Contributions should not:

- Rename core architectural components
- Redefine RiCo, REB, or CARE/REB
- Imply certification, compliance, approval, or legal authority
- Expose proprietary implementation logic
- Modify architectural meaning without explicit approval

---

## Review Standard

Architecture changes require explicit review and approval by Richard Colimon / ManChine AI Technology.

Case studies, examples, and scenario outputs may illustrate the architecture, but they do not modify the architecture.
