# Runtime Legitimacy

## Definition

Runtime legitimacy is the continuous demonstration that execution remains justified under present conditions.

Legitimacy is not inherited from system startup.

It must be maintained throughout execution.

---

## Why Legitimacy Matters

A system may:

- execute correctly,
- produce coherent outputs,
- satisfy technical requirements,

and still no longer possess a legitimate basis for continued action.

Correct execution is not sufficient.

Legitimate execution is required.

---

## Legitimacy vs Continuity

Execution continuity answers:

> Can the system continue?

Runtime legitimacy answers:

> Should the system continue?

These are different questions.

---

## Continuous Evaluation

Runtime legitimacy continuously evaluates:

- current authority
- current context
- current evaluability
- current consequence
- current governance state

Execution remains admissible only while legitimacy can still be demonstrated.

---

## Runtime Governance

Runtime governance exists to preserve legitimacy as conditions evolve.

Governance therefore becomes a continuous architectural function rather than a one-time policy review.

---

## Relationship to REB

The Runtime Execution Boundary (REB) evaluates legitimacy before consequential continuation.

Legitimacy is the governing condition.

REB is the enforcement boundary.
